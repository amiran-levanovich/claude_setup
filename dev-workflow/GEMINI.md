# dev-workflow — Gemini CLI entry point

Read [AGENTS.md](./AGENTS.md) in this directory and follow it — it is the canonical, harness-neutral instruction file for this kit. This wrapper exists only because Gemini CLI loads `GEMINI.md` by default; it adds nothing else. (Alternatively, set `contextFileName` to `AGENTS.md` in your Gemini settings and delete nothing — this file is inert then.)
