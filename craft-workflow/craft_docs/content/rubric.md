# Content — Acceptance Rubric

The "tests" for written work. Apply the **Always** section to everything; add the sub-mode sections that match the piece. Run every applicable dimension in the Step 5 review loop.

## Always (every piece)
- [ ] **Audience fit** — vocabulary, assumed knowledge, and framing match the reader in the brief.
- [ ] **One clear goal** — the piece drives the single desired action/takeaway; no competing asks.
- [ ] **Voice & tone** — consistent with the brand voice; no banned words; no unjustified jargon.
- [ ] **Truthful & grounded** — every factual claim is supported by the brief's source material. No invented stats, quotes, or features. Unknowns are flagged, not fabricated.
- [ ] **Tight** — no filler, no redundancy; every sentence earns its place. Cut throat-clearing intros.
- [ ] **Scannable** — meaningful headings, short paragraphs, front-loaded key points (inverted pyramid).

## Sub-mode: marketing / landing
- [ ] **Value proposition legible in ~5 seconds** — the hero answers "what is this, for whom, why care."
- [ ] **Benefits over features** — outcomes for the reader, not a spec list.
- [ ] **Customer language over company jargon** — words the audience actually uses.
- [ ] **One primary CTA**, repeated sensibly; the action is specific ("Start free trial", not "Submit").
- [ ] **Objection handling / social proof** present where the funnel needs it.

## Sub-mode: ux-microcopy (the 4 standards)
- [ ] **Purposeful** — every string serves a clear function; nothing decorative.
- [ ] **Concise** — minimal words without losing meaning; no "please"/"sorry" padding.
- [ ] **Conversational** — human and approachable; reads as one side of a helpful conversation.
- [ ] **Clear** — understood on first read; buttons name the action; errors say what happened + how to fix it.
- [ ] Patterns covered where relevant: button labels, error messages, empty states, form help, notifications.

## Sub-mode: seo
- [ ] **Search intent match** — the piece answers the actual intent behind the target query (informational/commercial/transactional), not just the keyword.
- [ ] **Primary keyword** in the title, H1, first ~100 words, and naturally throughout — no stuffing.
- [ ] **Heading hierarchy** — one H1; logical H2/H3 that map to sub-intents and are skimmable.
- [ ] **Metadata** — title tag (≤ ~60 chars) and meta description (≤ ~155 chars) written, compelling, keyword-bearing.
- [ ] **E-E-A-T** — demonstrates Experience, Expertise, Authoritativeness, Trust (credible sources, author rationale, accuracy).
- [ ] **Internal links / structured data** considered where applicable (schema for the content type).

### Judgment calls
When tone, length, or how hard to push the CTA is genuinely ambiguous, present options with trade-offs via AskUserQuestion rather than silently choosing. Never resolve a missing fact by inventing one — ask.
